# @fesjs/plugin-jest

集成 Jest在新窗口打开 测试框架，目前只支持单元测试和覆盖测试。

# 启用方式

在 package.json 中引入依赖：

{
    "dependencies": {
        "@fesjs/fes": "^2.0.0",
        "@fesjs/plugin-jest": "^2.0.0"
    },
}

# 约定

  • 项目根目录下 tests 和 __tests__ 文件夹中的 js 或者 jsx 文件为测试文件。
  • 需要覆盖测试的文件范围是src/index.{js,jsx,ts,tsx,vue}

例如测试文件 add.js：

fes-template
├── __tests__
│   └── add.js
└── src
    ├── .fes
    └── utils
        └── sum.js

# 配置

插件实现 Jest 的全部配置，具体请查看 文档-configuration在新窗口打开。 也可以使用 -h 打印配置帮助信息：

# args 变量

$ fes test -h
1

# 配置文件

除了插件内置的默认配置之外，插件遵循 Jest的配置文件规范，约定项目根目录下的 jest.config.js 为用户配置文件，约定 packages.json 的 jest 属性内容也是配置。

# 优先级

args 配置 > package.json中的 jest > jest.config.js > 默认配置
